# シナリオナビゲーター機能 設計書

## 概要

ダッシュボードヘッダー右端の「📘 シナリオ」ボタンから起動するStep by Stepガイド機能。EA Journey顧客がSPMポータルの操作手順を自己完結で学べるよう、画面上にガイドを常時表示する。

## UI構成

### 1. シナリオ選択モーダル

- フェーズ1・フェーズ2でグループ分け表示
- 各シナリオカードにタイトル・操作時間・ステップ数を表示
- クリックで選択し、サイドパネルを開く

### 2. サイドパネル（右端固定・300px幅）

| 要素 | 説明 |
|------|------|
| 進捗バー | 現在ステップ / 総ステップ数 |
| ❓ 問い | このステップで考えること |
| 💡 判断基準 | どう判断するか |
| 📌 理由 | なぜこの操作が必要か |
| ✅ やること | 具体的な操作手順 |
| 次へ・戻る | ステップナビゲーション |
| ⌄ 最小化 | 右下の丸タブに格納 |
| ✕ 閉じる | パネルを閉じてシナリオ終了 |

## シナリオデータ構造（JavaScript）

```javascript
var SCENARIOS = [
  {
    id: 's01',
    phase: 1,
    title: 'Scenario 01｜CSDMを理解する',
    steps: [
      {
        title: 'ステップタイトル',
        question: '❓ 問い',
        criteria: '💡 判断基準',
        reason: '📌 理由',
        todo: '✅ やること'
      }
    ]
  }
]
```

## 実装場所

- シナリオデータ：フロントエンドHTML内の `var SCENARIOS = [...]` に静的定義
- パネルHTML：同HTMLファイル内のモーダル・サイドパネルDOM
- 状態管理：JavaScript変数（`currentScenario`, `currentStep`）でローカル管理
- サーバーサイドAPIなし（全てフロントエンド完結）
